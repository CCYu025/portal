02 / FINDINGS

關鍵發現

從上述七維分析萃取出的可行動清單。標示為 Critical 的項目，建議在 Phase 1 即處理。

資料與視圖耦合

系統清單寫死在 HTML，新增一個系統等於改一塊 DOM。應抽離為 JSON 資料，由模板動態渲染。

無搜尋 / 無快捷鍵

當系統超過 10 個，眼掃就崩潰。需提供 ⌘K / Ctrl+K 全域搜尋 + 模糊比對。

無連線健康狀態

VPN 內網系統無法得知是否可達。應以 fetch + AbortController 做被動探測，前置綠/灰點。

缺乏個人化

無「最近使用 / 釘選」概念，每位工程師看到的是同一張無記憶頁面。應用 localStorage 紀錄使用頻率。

圖示佔位、不一致

目前卡片 SVG 各自獨立撰寫，難以統一維護。應導入單一 icon set（Lucide inline SVG）保持風格一致。

缺少 Landmark

無 <nav>、<aside>、<footer>，screen reader 無法跳區。修正為標準 HTML5 sectioning。

VPN 提示靜態

黃色提醒永遠顯示。可在偵測到能連線時自動轉為「已連線 ✓」綠色狀態。

主題系統紮實

CSS 變數已抽離得宜，重構成本低。新版可直接繼承 --bg / --accent / --surface。

03 / REFACTOR

重構建議（Before / After）

以下四個 Tab 涵蓋最關鍵的程式碼層級改動。所有改動皆相容單檔靜態部署。

將系統清單抽離為資料

把 DOM 換成可被 vanilla JS 渲染的 JSON 陣列，新增系統只需編輯一個物件。

  <div class="card-header">
    <h3>MES System V2</h3>

  <a href="http://100...">開啟</a>
</div>
/* 每個系統重複 ~10 行 */
AFTERdata.js (inline)
const SYSTEMS = [

⌘K 全域搜尋（純 vanilla JS）

按 Ctrl/⌘+K 開啟搜尋面板，輸入即時模糊比對名稱、描述、分類。

BEFORE—
// 不存在
// 使用者只能用 Ctrl+F 瀏覽器原生搜尋
// 但找到後仍需手動點擊按鈕
AFTERscript (inline)
document.addEventListener('keydown', e => {
  if ((e.metaKey||e.ctrlKey) && e.key==='k') {
    e.preventDefault();
    openPalette();
  }
});

function filter(q) {
  return SYSTEMS.filter(s =>
    [s.name,s.desc,s.category]
      .join(' ').toLowerCase()
      .includes(q.toLowerCase())
  );
}

被動健康檢查

用 fetch 配合 timeout，在卡片左上角顯示綠/灰圓點，VPN 提示自動變綠。

BEFORE—
// 永遠顯示「需公司 VPN」黃色橫幅
// 即使已連線也不會消失
// 使用者點下去才知道能不能用
AFTERscript
async function ping(url) {
  const ctrl = new AbortController();
  const t = setTimeout(()=>ctrl.abort(), 2000);
  try {
    await fetch(url, {
      mode:'no-cors',
      signal:ctrl.signal
    });
    return 'online';
  } catch { return 'offline'; }
  finally { clearTimeout(t); }
}
其他建議改動（展開）+
  • 字型自託管：把 Fira Sans 與 Noto Sans TC 的 woff2 檔放入 repo /fonts/，移除 Google Fonts 三個外部請求。
  • 釘選機制：右上角 ☆ 圖示，存 localStorage，置頂顯示「我的常用」分區。
  • 404 / 離線頁：提供 offline.html 供 GitHub Pages 部署為 SPA fallback。
  • i18n 結構：把所有文案抽到 I18N.zhHant 物件，預留 en / zhHans 鍵位。
不建議做的事（反模式）+

為了保住「純靜態、無建置」的核心優勢，不建議引入：

  • React / Vue / Svelte — 違背無建置原則。
  • Tailwind CLI — 需要 build step。
  • Service Worker 全離線 — 對內部入口而言，維護成本 > 效益。
  • Auth 機制 — 應由 VPN/Tailscale 層處理，非前端職責。

06 / ROADMAP

實施路線圖

分三階段交付，每階段皆為可獨立部署的 PR。預估總工時 6 週、每週 1 ~ 2 天投入。

PHASE 01 · FOUNDATION

地基重構：資料層 + 語意化

Week 1 – 2

把硬編碼的卡片轉為資料驅動，確保後續所有功能都有單一資料來源。同時補齊語意化與無障礙。

✓抽離 SYSTEMS JSON：建立 data/systems.js，包含 id/name/desc/url/category/icon/vpn 欄位。
✓實作 renderCards()：用 <template> + DocumentFragment 動態生成卡片。
✓整卡可點：把 div.card 改 a.card，移除多餘 .btn。
✓Landmark 與 aria：補 <nav>、aria-labelledby、skip link。
✓圖示系統：導入 Lucide inline SVG，icon 由 name 對映。
PHASE 02 · INTELLIGENCE

智能化：搜尋 + 狀態 + 個人化

Week 3 – 4

讓入口從「靜態清單」進化為「會回應」的工作台。每個工程師看到的頁面開始反映自身使用習慣。

✓⌘K 命令面板：覆蓋層 + 模糊比對 + 鍵盤上下選擇 + Enter 開啟。
✓健康檢查：fetch(no-cors) 配 2 秒 timeout，顯示綠/灰狀態點。
✓localStorage 使用頻率：每次點擊累計，「我的常用」分區自動排序。
✓釘選機制：卡片右上 ☆ 切換 pinned；釘選優先於使用頻率。
✓VPN 狀態自動偵測：透過任一 VPN URL 的 ping 結果，更新頂部狀態列。
PHASE 03 · POLISH

打磨：效能 + 細節 + 文件

Week 5 – 6

將整體完成度推至「可作為公司門面對外展示」的水準，並建立後續維運的文件基礎。

✓自託管字型：Fira Sans + Noto Sans TC woff2 收入 repo，移除外部請求。
✓進場動畫：卡片 IntersectionObserver staggered fade-in；尊重 prefers-reduced-motion。
✓404 / Offline 頁：一致風格的 fallback 頁面。
✓Lighthouse ≥ 95：四項分數線通過。
✓維運文件：README 寫明「如何新增一個系統」三步驟。

從入口頁到工作台，只需 6 週

本計畫所有改動皆可在 GitHub Pages 純靜態環境部署，不引入任何建置工具或框架，維持極低的維護成本。在第 1 階段結束時，入口頁就能支援數十個系統而不增加任何 DOM 負擔。
